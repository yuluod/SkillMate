# 安全策略

## 支持范围

SkillMate 目前处于早期版本阶段。安全修复会优先提供给最新发布版本；旧版本不保证单独维护。

## 私下报告漏洞

请通过仓库的 **Security → Report a vulnerability** 创建 GitHub Private Vulnerability Report。不要在公开 Issue、Discussion、PR 或社交平台披露尚未修复的漏洞。

报告请尽量包含：

- 受影响版本和操作系统
- 可复现步骤、最小样例及实际影响
- 涉及的路径、权限或信任边界
- 建议修复方式（如有）

维护者会尽力在 7 天内确认收到报告，并在验证后同步修复进展。修复发布前，请为用户保留合理的升级窗口。

## 安全边界

Skill 可能包含脚本、依赖声明、网络访问和环境变量引用。SkillMate 的静态检查仅用于辅助判断，不等同于恶意代码检测或安全审计；安装第三方 Skill 前仍应阅读其内容并确认来源可信。
